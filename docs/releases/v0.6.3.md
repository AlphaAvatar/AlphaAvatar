## AlphaAvatar v0.6.3 Release Notes

### Highlights

AlphaAvatar v0.6.3 introduces the first foundation for **graph-aware memory retrieval**.

This release refactors the Memory plugin around a cleaner `MemoryItem` model, multi-object ownership, graph node mentions, session-scoped local identity keys, and LanceDB-backed graph-aware retrieval. It prepares the memory system for future multimodal ENV memory extraction while keeping the current release focused on stable conversation/tool memory and graph retrieval infrastructure.

### What's New

* Added graph-aware `MemoryItem` support:

  * `object_ids` replaces single-object memory ownership.
  * Memory items can now carry `graph_nodes` and `graph_links`.
  * Memory content remains the source of truth, while graph nodes act as retrieval anchors.

* Added graph node mention extraction:

  * `PatchOp.node_mentions` lets the memory extractor provide lightweight graph anchors.
  * Graph nodes and graph links are built by runtime logic instead of being directly generated by the model.
  * Prompt output is simplified so `PatchOp.value` stays clean and human-readable.

* Added session-scoped local graph keys:

  * Local detector IDs such as `face:tmp_1`, `voice:speaker_0`, and `object:cup_1` are scoped by session.
  * This prevents cross-session identity pollution when the same temporary ID appears in different sessions.

* Added graph lookup infrastructure:

  * Added `GraphLookup` for alias resolution and graph neighbor expansion.
  * Added support for query-time alias fan-out instead of immediately rewriting the vector database.
  * Added 0-hop and optional 1-hop graph expansion support.

* Added LanceDB graph-aware retrieval:

  * LanceDB now stores both `memory_item` rows and `graph_node` occurrence rows.
  * Graph node rows are linked back to `memory_id`.
  * Added batch `node_keys` retrieval for exact graph-node lookup.
  * Added graph-node semantic retrieval.
  * Added object filtering using `object_ids`.

* Added MemoryBase graph APIs:

  * Added `search_by_graph_node()` as a public memory interface.
  * Added `save_graph_aliases()` as a future-facing identity/entity merge interface for Persona, ENV memory, channels, and other plugins.

* Updated memory extraction prompts:

  * Replaced turn-only wording with unified `SESSION CONTENT`.
  * Session content can include conversation, tool summaries, runtime context, and future ENV memory context.
  * Removed model-controlled `evidence` and `extra_data` from extraction output.
  * Evidence, object IDs, timestamps, session IDs, and runtime metadata are now owned by runtime logic.

### Notes

* ENV memory extraction is intentionally deferred to v0.6.4.
* Persona/identity merge hooks are not enabled yet; `save_graph_aliases()` is added as the interface foundation.
* Existing local LanceDB memory tables may need to be rebuilt because the schema changed from single `object_id` to `object_ids_json` and now includes graph-node occurrence rows.

### Internal Direction

v0.6.3 establishes the Memory plugin as a graph-aware retrieval layer:

* LLM extracts clean memory text and graph anchors.
* Runtime owns session scoping, evidence, object IDs, graph construction, and alias logic.
* LanceDB indexes both memory items and graph node occurrences.
* Future ENV memory can plug into the same memory item + graph node + retrieval path.
